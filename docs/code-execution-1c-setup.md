# Развёртывание подсистемы «ИИ» в УТ 10.3 (или другой обычной конфигурации)

Эта инструкция описывает создание трёх объектов в Конфигураторе вашей
**тестовой** копии УТ 10.3 (или другой конфигурации с обычными формами):

1. Подсистема `ИИ` (контейнер).
2. Регистр сведений `ИИ_Действия` (журнал действий).
3. Общий модуль `ИИ_Сервер` (функции выполнения кода).
4. Обработка `ИИ_Песочница` с обычной формой (UI для пользователя).

> **Предупреждение.** Все изменения делаются в основной конфигурации.
> После сохранения конфигурация будет считаться «изменённой» — типовые
> обновления с поставщика придётся накатывать в режиме сравнения/слияния.
> Рекомендуется делать это **только на копии** боевой базы, помеченной
> маркером `[Тест]` в названии информационной базы.

Время на развёртывание: **1.5 – 2 часа** разовой работы.

---

## 0. Подготовка

1. Откройте Конфигуратор → подключитесь к тестовой копии базы.
2. **Снимите конфигурацию с поддержки** если она ещё на поддержке:
   `Конфигурация → Поддержка → Настройка поддержки → Включить возможность изменения`.
   Для подсистем 1С это требуется, иначе вы не сможете добавить новые объекты.
3. Сделайте резервную копию через `Администрирование → Выгрузить информационную базу...`
   на всякий случай.

---

## 1. Подсистема `ИИ`

1. В дереве конфигурации правый клик на корне (название конфигурации)
   → `Подсистемы` → `Добавить`.
2. Свойства подсистемы:
   - **Имя:** `ИИ`
   - **Синоним:** `ИИ`
   - **Комментарий:** `Объекты для интеграции с ИИ-ассистентами через MCP-мост`
   - **Включать в командный интерфейс:** *да* (галка установлена).
3. Картинка подсистемы — необязательно, можно оставить стандартную.
4. Сохраните: `Файл → Сохранить` (Ctrl+S).

---

## 2. Регистр сведений `ИИ_Действия`

1. В дереве конфигурации: `Регистры сведений → Добавить`.
2. Свойства регистра:
   - **Имя:** `ИИ_Действия`
   - **Синоним:** `ИИ Действия`
   - **Комментарий:** `Журнал всех действий ИИ-ассистента в базе`
   - **Периодичность:** `Непериодический`
   - **Режим записи:** `Независимый`
   - **Основной отбор по периоду:** *нет*
3. На вкладке **Подсистемы** установите галку у `ИИ`.

### Измерения регистра

В дереве регистра: `Измерения → Добавить`.

**Измерение 1: ИдентификаторЗадания**
- Имя: `ИдентификаторЗадания`
- Тип: `УникальныйИдентификатор`
- Ведущее: *нет*
- Основной отбор: *да*

### Ресурсы регистра

В дереве регистра: `Ресурсы → Добавить`.

**Ресурс 1: Время**
- Имя: `Время`
- Тип: `Дата`, состав даты: `Дата и время`

**Ресурс 2: Пользователь**
- Имя: `Пользователь`
- Тип: `СправочникСсылка.Пользователи`
  *(если в вашей УТ нет этого справочника — используйте СправочникСсылка.Сотрудники
  или замените на Строку(100))*

**Ресурс 3: Комментарий**
- Имя: `Комментарий`
- Тип: `Строка`, длина: 0 (неограниченная), переменная длина

**Ресурс 4: КодПрямой**
- Имя: `КодПрямой`
- Тип: `Строка`, длина: 0, переменная длина

**Ресурс 5: КодОбратный**
- Имя: `КодОбратный`
- Тип: `Строка`, длина: 0, переменная длина

**Ресурс 6: СнимокДо**
- Имя: `СнимокДо`
- Тип: `Строка`, длина: 0, переменная длина

**Ресурс 7: СнимокПосле**
- Имя: `СнимокПосле`
- Тип: `Строка`, длина: 0, переменная длина

**Ресурс 8: Статус**
- Имя: `Статус`
- Тип: `Строка`, длина: 50, переменная длина
- Допустимые значения: `success`, `error`, `rolled_back`, `compile_error`,
  `runtime_error`, `rejected_by_safety`

**Ресурс 9: ТекстОшибки**
- Имя: `ТекстОшибки`
- Тип: `Строка`, длина: 0, переменная длина

**Ресурс 10: ДлительностьМс**
- Имя: `ДлительностьМс`
- Тип: `Число`, длина: 10, точность: 0

Сохраните.

## 3. Общий модуль `ИИ_Сервер`

1. В дереве: `Общие → Общие модули → Добавить`.
2. Свойства модуля:
   - **Имя:** `ИИ_Сервер`
   - **Синоним:** `ИИ Сервер`
   - **Комментарий:** `Серверные функции для выполнения кода через ИИ-мост`
   - **Глобальный:** *нет*
   - **Сервер:** *да* (галка)
   - **Клиент (управляемое приложение):** *нет*
   - **Клиент (обычное приложение):** *нет*
   - **Внешнее соединение:** *да* (галка) — это критично, чтобы COM-вызовы видели функции
   - **Вызов сервера:** *да*
   - **Привилегированный:** *нет* (по умолчанию)
3. На вкладке **Подсистемы** установите галку у `ИИ`.
4. Откройте модуль (двойной клик).
5. Скопируйте полностью содержимое файла
   [`onec/ИИ_Сервер.bsl`](../onec/ИИ_Сервер.bsl) в модуль.
6. Сохраните.

---

## 4. Обработка `ИИ_Песочница`

1. В дереве: `Обработки → Добавить`.
2. Свойства обработки:
   - **Имя:** `ИИ_Песочница`
   - **Синоним:** `ИИ Песочница`
   - **Комментарий:** `Чтение задач от ИИ из inbox-папки и их выполнение`
3. На вкладке **Подсистемы** установите галку у `ИИ`.

### Создаём форму обработки

1. В дереве обработки: `Формы → Добавить`.
2. Тип формы — **Форма обработки** (не «Форма списка»).
3. Имя формы: `Форма` (по умолчанию).
4. Назначение по умолчанию: *да*.
5. Запустится конструктор обычной формы — нажмите OK с дефолтными настройками.

### Реквизиты формы

В правой панели формы (свойства / реквизиты) добавьте:

| Имя реквизита | Тип | Назначение |
|---|---|---|
| `ПапкаДроппера` | Строка(500) | Путь к папке inbox |
| `АвтоРежим` | Булево | Флаг автоматического режима |
| `ИнтервалОпроса` | Число(3,0) | Секунды между опросами в авто-режиме |
| `СписокЗаданий` | ТаблицаЗначений | Колонки: ИмяФайла (Строка), Создано (Дата), Комментарий (Строка), ПутьФайла (Строка) |
| `Журнал` | ТаблицаЗначений | Колонки: Время (Дата), Комментарий (Строка), Статус (Строка), Длительность (Число), Идентификатор (УникальныйИдентификатор) |

### Элементы формы

Создайте элементы формы примерно такой структуры:

```
[Форма "ИИ Песочница"]
├── Группа "Настройки"
│   ├── Поле ввода:    ПапкаДроппера                    (длина 500, кнопка выбора каталога)
│   ├── Флажок:        АвтоРежим
│   └── Поле ввода:    ИнтервалОпроса                    (по умолчанию 5)
│
├── Группа "Inbox"
│   ├── ТабличноеПоле: СписокЗаданий                    (только чтение)
│   ├── Кнопка:        "Обновить"                        (вызывает ОбновитьСписокЗаданий)
│   ├── Кнопка:        "Выполнить выделенное"            (вызывает ВыполнитьВыделенное)
│   └── Кнопка:        "Выполнить всё"                   (вызывает ВыполнитьВсё)
│
└── Группа "Журнал"
    ├── ТабличноеПоле: Журнал                           (только чтение)
    ├── Кнопка:        "Обновить журнал"                 (вызывает ОбновитьЖурнал)
    ├── Кнопка:        "Откатить выделенное"             (вызывает ОткатитьВыделенное)
    ├── Кнопка:        "Просмотр кода"                   (вызывает ПросмотрКода)
    └── Кнопка:        "Повторить прямой код"            (вызывает ПовторитьПрямой)
```

### Модуль формы

1. Откройте модуль формы (двойной клик по форме → вкладка `Модуль`).
2. Скопируйте полностью содержимое файла
   [`onec/ИИ_Песочница_Форма.bsl`](../onec/ИИ_Песочница_Форма.bsl) в модуль.
3. Сохраните.

### Связь кнопок с процедурами

В обычных формах УТ 10.3:
- Каждая кнопка должна иметь **действие** (правый клик → Свойства → Действие).
- Действие — выбрать из списка или вписать имя процедуры из модуля формы.

Имена процедур, на которые вешать кнопки:
- `ОбновитьСписокЗаданийНажатие`
- `ВыполнитьВыделенноеНажатие`
- `ВыполнитьВсёНажатие`
- `ОбновитьЖурналНажатие`
- `ОткатитьВыделенноеНажатие`
- `ПросмотрКодаНажатие`
- `ПовторитьПрямойНажатие`

Также:
- Поле `АвтоРежим` → событие `ПриИзменении` → процедура `АвтоРежимПриИзменении`.

## 5. Сохранение и тест

1. `Файл → Сохранить` (Ctrl+S).
2. `Конфигурация → Обновить конфигурацию базы данных` (F7).
3. Откройте предприятие в режиме `1С:Предприятие`.
4. В командном интерфейсе должна появиться подсистема **ИИ**.
5. Откройте её → Обработки → **ИИ Песочница**.
6. Должна открыться форма с пустыми полями.
7. Заполните «Папка дроппера» — например `C:\Users\АдминистраторСбт\AppData\Local\1cMcpBridge\dropper\main`.
8. Нажмите «Обновить» — список должен быть пустым.
9. Нажмите «Обновить журнал» — должна показаться пустая таблица (если в регистре нет записей).

Если всё это работает — обработка готова к приёму заданий от Python-сервера.

---

## 6. Дальше

После того как 1С-сторона развёрнута:

1. В **1C Bridge Manager** в форме базы появится поле «Папка дроппера»
   (после установки v0.3.0). Туда указываете тот же путь что и в 1С.
2. Перезапускаете Claude Desktop.
3. В чате просите Claude выполнить какое-нибудь действие через `execute_code`.
4. Claude кладёт JSON в inbox → 1С обработка читает → выполняет → пишет
   результат в `done` и в регистр `ИИ_Действия`.
5. Если включить «Авто-режим» — обработка сама будет опрашивать inbox раз
   в N секунд.

## Приложение: что-то пошло не так

**«Объекты не отображаются в командном интерфейсе»** — проверьте что у каждого
объекта на вкладке «Подсистемы» установлена галка `ИИ`.

**«ИИ_Сервер.ВыполнитьЗадание не найдена»** — проверьте свойства модуля
`ИИ_Сервер`: должны быть включены `Сервер` и `Внешнее соединение`.

**«Не удаётся прочитать файл задания»** — проверьте права папки дроппера.
Папка должна быть доступна на чтение/запись пользователю, под которым работает 1С.

**«Регистр заполняется, но без пользователя»** — поправьте `ИИ_Сервер` в месте
где определяется текущий пользователь — у вас может быть другой справочник
пользователей.
