# 1C MCP Bridge

Подключение Claude Desktop напрямую к информационной базе 1С:Предприятие через
[Model Context Protocol](https://modelcontextprotocol.io/). Claude получает четыре
инструмента (`execute_query`, `describe_object`, `list_metadata`,
`get_object_by_ref`) и работает с базой как с обычным источником данных:
формулирует запрос на языке запросов 1С, исполняет, возвращает результат, считает
аналитику.

```
┌──────────────────┐   stdio    ┌──────────────────┐    COM    ┌──────────────────┐
│  Claude Desktop  │ ─────────► │ MCP-сервер (Py)  │ ────────► │ 1С:Предприятие   │
└──────────────────┘            └──────────────────┘           └──────────────────┘
```

## Кому это нужно

* Бухгалтеру / финансовому аналитику, который хочет сказать «покажи топ-10
  покупателей за апрель» и получить таблицу.
* Разработчику 1С, которому удобно проверить структуру конфигурации, не лазая в
  Конфигуратор.
* Команде, у которой типовая на поддержке (УТ 10.3, УТ 11, БП 3.0, ERP, ЗУП) —
  ничего в конфигурации менять не нужно, все запросы только на чтение.

## Что поддерживается

| Среда | Статус |
|---|---|
| Платформа 1С:Предприятие | 8.2, 8.3, 8.5 |
| Файловая база (.1CD) | да |
| Клиент-серверная база (1С-сервер) | да |
| Аутентификация Windows (OS-auth) | да |
| Аутентификация средствами 1С | да |
| ОС | Windows 10 / 11 / Server 2016+ |

## Установка

1. Скачать `1cMcpBridge-Setup-<версия>.exe` со страницы
   [Releases](../../releases).
2. Запустить от имени администратора (нужно для регистрации COM-коннектора).
3. Пройти 5 шагов мастера:
   1. Принять лицензию.
   2. Выбрать установленную версию платформы 1С (определяется автоматически).
   3. Указать тип ИБ — файловая или клиент-серверная.
   4. Ввести параметры подключения и (при желании) проверить кнопкой «Проверить
      подключение к 1С».
   5. Подтвердить установку.
4. После установки — перезапустить Claude Desktop (правый клик по иконке в
   трее → Quit, потом запустить заново). В новом чате внизу появится индикатор
   с MCP-сервером `1c-bridge` и четыре инструмента.

Дальше задавайте вопросы:

> Покажи топ-10 покупателей по выручке за апрель 2026 с использованием регистра
> Продажи. Сначала посмотри его структуру через describe_object.

Claude сам пройдёт по инструментам, составит и выполнит запрос, вернёт таблицу.

## Безопасность

* Все запросы идут только на чтение — `execute_query` использует объект
  `Запрос` платформы, который не модифицирует данные.
* Учётные данные базы хранятся локально в `%APPDATA%\Claude\claude_desktop_config.json`
  и никогда не уходят за пределы компьютера. Anthropic видит только текст
  запроса и табличный результат — ровно то, что вы решите отправить как часть
  вопроса Claude.
* Рекомендуется завести в 1С отдельного пользователя «AI-аналитик» с правами
  только на просмотр нужных объектов.

## Сборка из исходников

Нужны: Windows, [Inno Setup 6](https://jrsoftware.org/isdl.php), Python 3.12.

```powershell
git clone https://github.com/dimax27/1c-mcp-bridge.git
cd 1c-mcp-bridge
$env:APP_VERSION = '0.1.0'
& "${env:ProgramFiles(x86)}\Inno Setup 6\ISCC.exe" installer\setup.iss
# Готовый .exe появится в .\dist\
```

Или просто нажать `Run workflow` на вкладке Actions — GitHub соберёт всё сам.

## Диагностика проблем

Если что-то не работает — запусти диагностику:

```powershell
powershell -ExecutionPolicy Bypass -File "C:\Program Files\1cMcpBridge\installer\detect_1c.ps1"
```

Покажет какие платформы 1С нашлись, зарегистрированы ли коннекторы, есть ли
Python и Claude Desktop.

Логи установки лежат в `C:\Program Files\1cMcpBridge\installer\install.log`.

Логи работы MCP-сервера видны в Claude Desktop:
`%APPDATA%\Claude\logs\mcp-server-1c-bridge.log`.

Подробнее — см. [docs/troubleshooting.md](docs/troubleshooting.md).
